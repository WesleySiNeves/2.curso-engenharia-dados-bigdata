Aula: Data Warehouse, Data Mart, Data Lake e Data Mesh

Este material foi criado para ajudá-lo a entender os conceitos de Data Warehouse, Data Mart, Data Lake e Data Mesh, com explicações teóricas, exemplos práticos e um mapa mental interativo para facilitar sua preparação para o concurso público.

1. Data Warehouse

Um Data Warehouse é um repositório centralizado que armazena grandes quantidades de dados históricos de várias fontes, organizados para suportar análises e relatórios. Ele é otimizado para consultas analíticas (OLAP) e geralmente usa modelagem dimensional (como esquemas em estrela ou floco de neve).

  • Características: Dados estruturados, integração de múltiplas fontes, otimizado para leitura, histórico de dados.
  • Uso: Relatórios empresariais, análises de tendências, suporte à tomada de decisão.

Exemplo:

Uma rede de varejo usa um Data Warehouse para consolidar dados de vendas de todas as lojas, permitindo análises como "Qual produto vendeu mais em cada região no último trimestre?".

2. Data Mart

Um Data Mart é um subconjunto do Data Warehouse, focado em um departamento ou área específica da organização (ex.: marketing, finanças). Ele é menor, mais especializado e projetado para atender necessidades específicas.

  • Características: Dados estruturados, escopo limitado, alta performance para análises específicas.
  • Uso: Relatórios departamentais, análises rápidas para equipes específicas.

Exemplo:

O departamento de marketing de uma empresa cria um Data Mart com dados de campanhas publicitárias para analisar o retorno sobre investimento (ROI) de cada campanha.

3. Data Lake

Um Data Lake é um repositório centralizado que armazena dados brutos, estruturados, semiestruturados e não estruturados, em seu formato nativo. É ideal para análises avançadas, como aprendizado de máquina e big data.

  • Características: Flexibilidade de formatos, escalabilidade, dados brutos, requer governança.
  • Uso: Análises preditivas, ciência de dados, integração com ferramentas de big data.

Exemplo:

Uma empresa de tecnologia armazena logs de servidores, dados de sensores IoT e postagens de redes sociais em um Data Lake para análises de comportamento do usuário com machine learning.

4. Data Mesh

O Data Mesh é um paradigma descentralizado que trata os dados como produtos, distribuindo a responsabilidade pela gestão de dados entre equipes (domínios). Ele promove autonomia, interoperabilidade e governança distribuída.

  • Características: Descentralização, dados como produtos, foco em domínios, interoperabilidade.
  • Uso: Organizações complexas com múltiplas equipes, escalabilidade de governança de dados.

Exemplo:

Uma empresa global permite que cada unidade de negócios (ex.: vendas, logística) gerencie seus próprios dados como "produtos", com padrões comuns para integração com outras equipes.
