🏢 DataWarehouse (DW)

📖 Definição

Um DataWarehouse é um repositório centralizado de dados integrados de múltiplas fontes, otimizado para análise e relatórios. Criado por Bill Inmon nos anos 1990.

🎯 Características Principais

• Orientado por Assunto: Organizado por temas de negócio

• Integrado: Dados de diferentes fontes consolidados

• Não-volátil: Dados históricos preservados

• Variante no Tempo: Mantém histórico temporal

💼 Exemplo Prático - Banco

Cenário: Um banco precisa analisar dados de clientes, empréstimos e investimentos.

Fontes: Sistema de contas correntes, sistema de empréstimos, sistema de investimentos

DataWarehouse: Integra todos os dados com histórico temporal para análises de risco e rentabilidade

🏪 DataMart

📖 Definição

Um DataMart é um subconjunto especializado de um DataWarehouse, focado em uma área específica de negócio ou departamento.

🎯 Tipos de DataMart

• Dependente: Deriva de um DataWarehouse existente

• Independente: Criado diretamente das fontes

• Híbrido: Combina dados do DW e fontes externas

💼 Exemplo Prático - Varejo

DataWarehouse: Contém todos os dados da empresa

DataMart de Vendas: Apenas dados de vendas, produtos e clientes

DataMart de RH: Apenas dados de funcionários e folha de pagamento

Vantagem: Acesso mais rápido e focado para cada departamento

🔍 DataMart vs DataWarehouse

Escopo: DataMart é específico, DW é corporativo

Tamanho: DataMart é menor, DW é maior

Usuários: DataMart atende um departamento, DW atende toda empresa

Implementação: DataMart é mais rápido de implementar

🌊 DataLake

📖 Definição

Um DataLake é um repositório que armazena grandes volumes de dados em seu formato nativo, incluindo dados estruturados, semi-estruturados e não-estruturados.

🎯 Características Principais

• Schema-on-Read: Estrutura definida na leitura

• Flexibilidade: Aceita qualquer tipo de dado

• Escalabilidade: Armazena petabytes de dados

• Custo-efetivo: Armazenamento mais barato

🕸️ DataMesh

📖 Definição

O DataMesh é uma arquitetura descentralizada que trata dados como produtos, com propriedade distribuída por domínios de negócio. Conceito criado por Zhamak Dehghani.

🎯 Quatro Princípios Fundamentais

1. Propriedade por Domínio: Cada área possui seus dados

2. Dados como Produto: Tratamento com mentalidade de produto

3. Plataforma Self-Service: Infraestrutura automatizada

4. Governança Federada: Padrões globais, execução local
